# Web3 行业岗位划分（系统化说明）

**日期：** 2026-05-23  
**标签：** #career #web3 #roles

## 背景 / 要解决什么问题

Web3 招聘常混用「区块链工程师」「DeFi 开发」「Web3 全栈」等头衔，职责边界不清。本文按 **行业层次 → 职能条线 → 典型岗位** 建立一张地图，便于选型学习、投递与团队分工。

---

## 一、先看三层：价值在哪里产生

```text
┌─────────────────────────────────────────────────────────────┐
│  应用层 Application     钱包 / 交易所 App / 链游 / 社交 / RWA 前端  │
├─────────────────────────────────────────────────────────────┤
│  协议层 Protocol        L1/L2、DeFi 协议、NFT、账户抽象、跨链桥   │
├─────────────────────────────────────────────────────────────┤
│  基础设施 Infrastructure 节点、索引、安全、托管、数据、开发工具链    │
└─────────────────────────────────────────────────────────────┘
         ▲ 横切：安全审计、经济模型、合规、运营增长、投资研究
```

| 层次 | 典型产物 | 岗位更偏 |
|------|----------|----------|
| 基础设施 | 客户端、RPC、区块浏览器、The Graph、Hardhat/Foundry | 平台工程、DevOps、底层协议研发 |
| 协议层 | 智能合约、共识/执行升级、代币与治理 | 智能合约、协议研究、密码学/共识 |
| 应用层 | dApp、CEX/DEX 产品、GameFi、支付 | 全栈 / 前端、产品、增长 |

多数 **入门岗位** 落在「协议层合约 + 应用层前后端」；**基础设施** 门槛更高，多要求系统编程或长期运维经验。

## 二、按职能条线划分（8 大类）

### 1. 智能合约 / 链上开发（On-chain Engineering）

**职责：** 设计并实现链上逻辑（Solidity / Vyper / Rust-for-SVM 等），保证安全、可升级与 Gas 效率。

| 细分岗位 | 主要工作 | 常见技能 |
|----------|----------|----------|
| 智能合约工程师 | 业务合约、库、测试、部署脚本 | Solidity、Foundry/Hardhat、OpenZeppelin |
| 协议工程师 | AMM、借贷、质押、治理合约体系 | DeFi 机制、经济模型对接、形式化验证入门 |
| 合约架构师 | 模块划分、升级策略、权限与多签 | Proxy 模式、访问控制、审计配合 |
| L2 / Rollup 工程师 | Sequencer、证明、桥合约 | OP Stack、zk 栈、Go/Rust |

**与本仓库：** [`notes/hardhat/`](../hardhat/)、[`projects/hardhat-learning`](../../projects/hardhat-learning)。

---

### 2. 链下后端 / 索引 / 数据（Off-chain & Data）

**职责：** 为 dApp 提供 API、监听链上事件、维护数据库与缓存，**不替代**链上真相来源。

| 细分岗位 | 主要工作 | 常见技能 |
|----------|----------|----------|
| 区块链后端工程师 | 交易构造、Nonce、多链 RPC、钱包对接 | Node/Go、ethers.js/viem、PostgreSQL |
| 索引 / 子图工程师 | 事件解析、GraphQL、历史数据 | The Graph、Subsquid、ClickHouse |
| 数据工程师 | 链上分析、标签、看板 | Dune、BigQuery、Python |
| MEV / 搜索者相关研发 | 捆绑、套利路径（合规内） | Go、低延迟、链上模拟 |

---

### 3. 前端 / 全栈 / 钱包体验（dApp & Wallet UX）

**职责：** 连接钱包、签名、交易状态、多链切换；把复杂链上操作做成可用产品。

| 细分岗位 | 主要工作 | 常见技能 |
|----------|----------|----------|
| Web3 前端工程师 | dApp UI、wagmi/ethers、SIWE 登录 | React/Next、TypeScript、WalletConnect |
| 全栈工程师 | 前后端 + 合约联调 | 同上 + 后端 + Hardhat |
| 钱包客户端工程师 | 扩展 / 移动端、密钥与安全 UI | React Native、安全存储、EIP-1193 |
| 账户抽象（AA）工程师 | Paymaster、Bundler、智能账户 | ERC-4337、Stackup/Pimlico 等 |

---

### 4. 安全（Security）

**职责：** 在上线前发现漏洞；事故后溯源；建立安全流程。

| 细分岗位 | 主要工作 | 常见技能 |
|----------|----------|----------|
| 智能合约审计师 | 手工审计 + 工具（Slither、Aderyn） | Solidity 深度、常见漏洞模式 |
| 安全研究员 | 0-day、桥攻击分析、披露 | 逆向、Foundry fuzz、链上取证 |
| 安全工程师（AppSec） | 交易所 / 托管内部系统 | 传统安全 + HSM、密钥 ceremony |
| 红队 / 赏金猎人 | 公开赏金、竞赛 | Immunefi、Code4rena |

---

### 5. 协议研究 / 经济与设计（Research & Tokenomics）

**职责：** 机制设计、参数与激励；与工程对齐「能否实现、是否可审计」。

| 细分岗位 | 主要工作 | 常见技能 |
|----------|----------|----------|
| 协议研究员 | 竞品分析、提案、治理论坛 | 经济学、DeFi 数学、英文写作 |
| 代币经济学家 | 排放、锁仓、博弈 | 激励设计、模拟（cadCAD 等） |
| 治理运营 | 提案流程、委托、社区投票 | Snapshot、Tally、社区沟通 |

---

### 6. 基础设施与平台（Infrastructure & Platform）

**职责：** 让链和开发者「跑得稳、用得省」。

| 细分岗位 | 主要工作 | 常见技能 |
|----------|----------|----------|
| 节点 / DevOps | 验证者、RPC 节点、监控 | Linux、K8s、Prometheus |
| 核心协议开发（L1） | 客户端、共识、执行层 | Go/Rust、以太坊/ Cosmos 代码库 |
| 开发者关系（DevRel） | 文档、示例、黑客松 | 工程 + 演讲、SDK |
| 区块链平台工程 | 内部链、联盟链、企业链 | Fabric/Besu、权限模型 |

---

### 7. 产品、运营、增长（Product & Growth）

**职责：** 定义做什么链上功能、如何获客与合规落地；不写合约但要懂约束。

| 细分岗位 | 主要工作 | 常见技能 |
|----------|----------|----------|
| Web3 产品经理 | PRD、链上/链下拆分、里程碑 | 竞品、Gas/UX 权衡 |
| 社区 / 运营 | Discord、大使、活动 | 内容、英文、危机沟通 |
| 增长 / BD | 生态合作、上币、集成 | 商务、数据分析 |
| 合规 / 法务（Crypto） | KYC、牌照、条款 | 各国监管框架 |

---

### 8. 投资、交易、做市（Markets — 非纯技术但常 adjacent）

| 细分岗位 | 主要工作 | 备注 |
|----------|----------|------|
| 研究员（基金） | 赛道 mapping、尽调 | 偏金融与叙事 |
| 量化 / 做市开发 | 策略、执行、风控 | 传统 quant + 链上延迟 |
| 交易员 / 投资经理 | 仓位与组合 | 牌照机构内岗位 |

技术转岗者常从 **工程 → 研究/安全** 横向移动，而非直接进入交易岗。

## 三、按赛道（垂直行业）再看岗位侧重

同一职能在不同赛道 **日常差别很大**：

| 赛道 | 合约侧重 | 后端/数据 | 前端侧重 |
|------|----------|-----------|----------|
| **DeFi** | AMM、清算、预言机 | 价格、仓位、风控 API |  swap、仓位、收益展示 |
| **NFT / 游戏** | ERC-721/1155、随机数、市场 | 元数据、撮合、链下匹配 | 铸造、市场、钱包登录 |
| **L2 / 跨链** | 桥、消息传递 | 充值提现状态机 | 跨链 UI、网络切换 |
| **RWA / 支付** | 合规托管接口、限额 | KYC 对接、法币通道 | 简化签名流程 |
| **DAO / 社交** | 治理、身份、声誉 | 索引、通知 | 投票、Feed |
| **基础设施 / 钱包** | 较少业务合约 | RPC、Bundler | SDK、文档 |

## 四、岗位对照：头衔易混淆

| 招聘常写 | 实际更接近 | 辨别问题 |
|----------|--------------|----------|
| 区块链开发工程师 | 合约 **或** 后端 **或** 全栈 | 是否写 Solidity？是否部署节点？ |
| Web3 全栈 | 前端 + 合约 + 简单后端 | JD 里三块占比各多少？ |
| DeFi 开发 | 协议合约 + 部分后端 | 是否涉及经济参数调优？ |
| 区块链架构师 | 合约架构 / 多链方案 / 企业链 | 偏业务还是偏基础设施？ |
| 元宇宙开发 | 常 = 游戏客户端 + NFT | 是否真上链？哪条链？ |

面试或选课时用这三问快速定位：**主写合约吗？主对接 RPC/索引吗？主做用户界面吗？**

## 六、典型成长路径（技术向）

```text
路径 A（合约为主）
  入门：Solidity + Hardhat/Foundry + 测试网部署
    → 中级：DeFi/NFT 业务合约 +  gas 优化 + 基础安全
      → 高级：升级/治理/跨链 + 审计协作 + 形式化验证

路径 B（全栈 dApp）
  入门：前端钱包连接 + 简单合约
    → 中级：索引后端 + 多链 + AA
      → 高级：性能、风控、复杂状态同步

路径 C（安全）
  入门：合约开发 + CTF（Ethernaut 等）
    → 中级：审计实习 / 赏金低危
      → 高级：独立审计、协议级研究
```

非技术路径：**运营 → 社区负责人 → 生态负责人**；**研究助理 → 协议研究员**。

## 七、与求职 / 学习的对应建议

1. **先定层次**：更想做链上逻辑（协议层）还是产品界面（应用层）？  
2. **再定赛道**：DeFi / 游戏 / 基础设施等决定项目作品集方向。  
3. **作品集**：合约岗 → 开源协议 + 测试 + 部署记录；全栈 → 完整 dApp + README。  
4. **避免只堆证书**：链上可验证贡献（GitHub、审计报告、赏金）权重高于培训证。  
5. **与本仓库**：合约基础走 Hardhat 模块；岗位地图走本文件；外部课程见 [`docs/learning-resources.md`](../../docs/learning-resources.md)。

## 要点小结

1. Web3 岗位可按 **基础设施 / 协议 / 应用** 三层 + **八类职能** 理解，不必被单一头衔迷惑。  
2. **智能合约、链下后端、前端钱包、dApp 安全** 是技术岗四条主干，协作紧密。  
3. **研究 / 产品 / 运营** 决定「做什么」，工程决定「怎么做、是否安全」。  
4. 选岗时用「合约 / 后端 / 前端」三问拆解 JD，再对照垂直赛道。  
5. 入门技术岗最常见组合：**Solidity + 测试框架 + 一种前端钱包栈 + 基础安全常识**。
