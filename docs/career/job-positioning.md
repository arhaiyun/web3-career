## 不建议作为主线的方向

- **纯 Solidity 合约工程师**：会和 2-4 年 Solidity 专职工程师竞争，不能最大化你的风控和 AI Agent 优势。
- **纯前端 dApp 工程师**：与履历主线偏离，除非作为作品集展示能力。
- **纯量化交易员**：需要交易策略和金融市场业绩，不是当前最强迁移路径。
- **纯运营 / BD / 社区岗位**：可作为行业理解补充，不作为技术主线。

## 简历版本使用规则

1. **大厂 / AI 公司 / 内容安全 / 平台架构岗位**：使用 `resume-ai-architect.md`。
2. **交易所 / 钱包 / 链上数据 / 合规科技 / Web3 安全岗位**：使用 `resume-web3-risk-ai.md`。
3. **猎头初筛**：先给大厂 AI 架构版，除非对方岗位明确是 Web3 风控或合规。
4. **技术负责人推荐**：附带 3 个面试故事标题和 1 个系统设计主题，让对方快速判断资深度。

## 谈薪策略

- 薪酬信息不写入简历，只作为内部谈判锚点。
- 主叙事不是“我要转型所以降维”，而是“AI Agent + 风控平台能力进入新场景”。
- 对 Web3 岗位要区分现金薪酬、Token、锁定期、合规雇佣实体和远程时区成本。
