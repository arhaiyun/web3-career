## 作品 1：链上地址风险画像看板

**目标岗：** On-chain Risk Intelligence · Crypto Fraud Risk  
**关联学习：** `onchain-data` 轨道

### 2 分钟口述稿

> 这个项目解决的是调查员面对 **裸地址** 时信息不足的问题。输入一个地址，看板聚合 **标签**（交易所、混币、诈骗关联）、**3-hop 资金流**、**风险分** 和 **人工复核历史**。
>
> 架构上：链上数据通过索引 API 拉取，标签层是我定义的 v0.1  taxonomy——按 **实体类型、风险等级、置信度、来源** 四元组存；图谱用简版邻接表或图 DB，支持调查员展开上下游。风险分不是黑盒一个数，而是 **可展开的规则+模型分项**，方便申诉和审计。
>
> 我刻意做了 **误伤提示**：当地址同时命中「交易所热钱包」和「高风险」时，界面要求二次确认。这来自小红书证据链经验—— **展示依据比展示结论更重要**。
>
> 局限：标签覆盖率依赖第三方；链上隐私链演示不完整。下一步接 **案件 ID** 和 Compliance Copilot 输出联动。
>
> 这个作品证明我能把 **链上数据翻译成风控决策输入**，而不是做链上数据分析 alone。

### 5 个追问

1. 标签冲突怎么 resolve？  
2. 3-hop 性能怎么优化？  
3. 和 Chainalysis/Elliptic 类数据如何分工？  
4. 风险分更新频率？  
5. 如何评估误标率？

## 作品 2：Crypto 提现风控规则引擎

**目标岗：** Crypto Fraud Risk · Web3 Risk AI Architect  
**关联学习：** `exchange-risk` 轨道

### 2 分钟口述稿

> 提现是 CEX **资损和监管** 的交汇点。我设计了一条 **四类信号** 融合的决策链：**账户/KYC、设备、行为、链上地址**。规则引擎输出 **拦截 / 增强验证 / 人工复核 / 放行** 四档，全程 **审计日志** 可回放。
>
> v0.1 包含约 12 条规则，例如：新设备 + 大额 + 高风险地址标签 → 人工；Sybil 模式行为分 → 增强验证。规则 DSL 支持 **优先级和短路**，避免低优先级规则掩盖高优紧急规则。
>
> 人工复核台字段对齐小红书审核台思路：**结论、依据、相似历史案件、升级路径**。指标看 **拦截命中率、误伤申诉率、平均处置时长**。
>
> 我借鉴阿里 RMB **实时反作弊**：高峰 QPS 下仍要保证决策 RT；链上特征 miss 时要有 **降级策略**（更保守 or 更依赖人工，需产品定）。
>
> 局限：尚未接真实链上 API；规则未做灰度实验框架。这是我最贴近 **CEX 风控岗** 的作品。

### 5 个追问

1. 规则与 ML 分数如何组合？  
2. VIP 用户误拦怎么办？  
3. 链上地址延迟 30s 怎么处理？  
4. 审计日志保留与合规？  
5. 和 Travel Rule 节点如何交互？

## 作品 3：AI 合规调查助手（Risk Copilot）

**目标岗：** Compliance AI Agent · Web3 Risk AI Architect  
**关联学习：** `ai-agent-risk` + `notes/dify/`

### 2 分钟口述稿

> 调查员输入 **案件 ID 或地址上下文**，Copilot 输出：**案情摘要、风险解释、缺失证据清单、建议下一步动作**——不输出「冻结/上报」等不可逆指令。
>
> 架构：Orchestrator + **工具白名单**（只读链上查询、内部 KYC 摘要、历史 case RAG）+ **结构化 JSON 输出** 写入案件系统。护栏五条红线：不自动 SAR、不自动冻结、不跨权限查 PII、不引用未验证链外 OSINT、低置信度必须标 human-review。
>
> 评估：离线 **案件摘要 BLEU/人工评分** + 线上 **调查员编辑率、采纳率、案件周期**。和小红书 Agent 审核共用思路：**评估集来自真实脱敏案件**。
>
> 我在 Dify 搭了最小 workflow 验证摘要质量；生产形态应对接权限与审计。
>
> 这个作品衔接 **#1 Agent 审核** 和 **#2 证据链**，面向 AML 调查而非内容 Moderation。

### 5 个追问

1. RAG 案件库如何脱敏？  
2. 工具调用被 prompt injection 怎么办？  
3. 与 LLM 网关的成本控制？  
4. 监管能否接受 AI 摘要作为证据？  
5. 和红队测试如何结合？
