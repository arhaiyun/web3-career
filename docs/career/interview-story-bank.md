# 面试故事库（STAR）

目标：把经历转成可在技术面、架构面、行为面复用的故事。每个故事都要能讲 2 分钟、5 分钟、15 分钟三个版本。

## 故事 1：小红书新一代 Agentic AI 审核系统

- **适用岗位**：AI Agent 工程负责人、AI 应用架构师、Trust & Safety AI
- **Situation**：内容安全审核覆盖笔记、评论、直播、私信、账号、广告、交易等多业务，传统审核链路依赖黑盒模型与人工复核，解释性、证据链、效率和成本都有提升空间。
- **Task**：作为内容安全 AI 审核平台 POC / Owner，带领 3-5 人建设新一代 Agentic AI 审核系统。
- **Action**：设计 Agent Orchestrator、RAG、工具调用、证据链服务、模型网关、评估平台、审核工作台；实现审核辅助、全链路还原分析、策略生成和评估闭环。
- **Result**：线上跑通完整流程；审核新模式在生态、交易、安全上线；2025 年至今笔记/评论/账号共节省约 1.32 亿成本，笔记人审率 7% -> 2.86%，机审自动处置占比 92.3% -> 95.05%。
- **追问准备**：Agent 如何编排？工具如何定义？RAG 检索什么？幻觉如何控制？评估集怎么构建？人工复核如何兜底？

## 故事 2：审核白盒化与电子证据链

- **适用岗位**：Trust & Safety AI、LLM Safety、内容安全平台架构师
- **Situation**：内容审核需要解释“为什么判定”，黑盒模型结果难以支持策略复盘、申诉、质检和运营理解。
- **Task**：推动审核决策从黑盒化转向白盒化、证据化、可解释化。
- **Action**：搭建证据链服务，沉淀模型输出、规则命中、上下文、用户行为、相似案例、策略依据；将电子证据铺到生态安全全队列。
- **Result**：审核链路可解释性提升，支持策略生成、全链路还原、申诉复核和质检闭环；触达发信申诉率 4.68% -> 3.64%，环比下降 22.22%。
- **追问准备**：证据粒度如何设计？证据可信度如何评估？证据链如何服务申诉？如何避免“解释看似合理但事实错误”？

## 故事 3：大模型审核能力规模化落地

- **适用岗位**：AI 应用架构师、LLM 平台工程、AI Agent 工程负责人
- **Situation**：内容安全业务需要在多个复杂场景中引入大模型能力，但不能只停留在 Demo。
- **Task**：让大模型能力在真实业务队列中可控、可评估、可规模化。
- **Action**：接入评论导流/营销、营销巡检、天眼、账号首页导流等业务；建设模型网关、策略编排、评估体系和人工兜底。
- **Result**：大模型审核能力在多业务线落地并拿到规模收益，交易体裁新模式日均增益免审 2w+。
- **追问准备**：模型选型标准？成本与延迟怎么控制？评估指标有哪些？灰度策略怎么做？

## 故事 4：小红书内容审核人审率大幅下降

- **适用岗位**：内容安全平台、风控平台、AI 应用架构
- **Situation**：内容审核人力成本高，且业务规模持续增长，必须提升机审自动处置能力。
- **Task**：在保证准召和用户体验的前提下降低人审率。
- **Action**：通过新审核模式、复杂规则引擎、策略平台、模型能力、证据链和评估闭环优化审核链路。
- **Result**：笔记人审率从 7% 降至 2.86%，机审自动处置占比从 92.3% 提升至 95.05%；生态笔记人审量 62.31w -> 29.93w，评论 74.34w -> 30.69w。
- **追问准备**：降人审如何避免误伤？准召怎么平衡？哪些场景必须保留人工？如何定义年度目标？

## 故事 5：阿里 ODS 风控数据服务千万级 QPS

- **适用岗位**：风控平台架构师、高并发平台、Staff Engineer
- **Situation**：风控中台需要快速、稳定地补全各种 KV 风险数据，并屏蔽底层存储差异。
- **Task**：担任 ODS 数据服务系统 Owner，保障大促高峰稳定性和低延迟。
- **Action**：设计多级缓存、多存储适配、在线/离线同步、异地集群同步、预热和统一服务层。
- **Result**：名单匹配峰值 1600w+ QPS，日常 600w+ QPS；风险识别名单请求 RT 控制在 0.2ms 内，日常均值低于 0.15ms；HSF 调用到服务端响应保持在 1ms 内。
- **追问准备**：缓存层级怎么设计？热点 key 怎么处理？一致性和延迟如何权衡？降级策略是什么？

## 故事 6：阿里营销反作弊从 0 到 1

- **适用岗位**：风控平台、反作弊、Crypto Fraud Risk
- **Situation**：集团营销活动涉及权益投放，容易被羊毛党、黑产和异常账号刷取，造成平台和商家资损。
- **Task**：作为 RMB 技术负责人，建设营销反作弊系统并保障大促。
- **Action**：从 0 到 1 搭建权益反作弊平台、行为风控风险感知模块、实时策略和稳定性保障链路。
- **Result**：支撑双十一 50w+ QPS；春晚红包雨 150w QPS * 10min * 4 次；日常 RT 13ms，超时率十万分之一内。
- **追问准备**：羊毛党特征有哪些？规则和模型怎么协同？如何设计实时决策？如何做大促压测？

## 故事 7：从传统风控迁移到 Web3 Risk AI

- **适用岗位**：Web3 Risk AI、Crypto Fraud、Compliance Agent
- **Situation**：Web3 风控需要同时理解账户风险、链上地址、交易行为、钓鱼诈骗、AML/KYT 和人工复核。
- **Task**：将阿里活动反作弊、小红书内容安全 AI 和 Agent 审核经验迁移到 Crypto 风控。
- **Action**：抽象出账号/设备/行为/内容/证据链/人工复核通用模型，并映射到 CEX 提现、空投 Sybil、钓鱼链接、链上地址标签和合规案件调查。
- **Result**：形成 Web3 Risk AI 简历版本、3 个作品集方向和 8 周准备计划。
- **追问准备**：Web3 与传统风控最大差异？链上地址如何入策略？KYT 和内容安全怎么结合？

## 故事 8：AI Agent 评估体系建设

- **适用岗位**：LLM Evaluation、AI Platform、AI Safety
- **Situation**：Agent 输出不能只看“是否像答案”，必须在真实审核场景中可度量、可回归、可上线。
- **Task**：建设支持审核辅助和策略生成的评估体系。
- **Action**：围绕业务场景、风险信号、策略动作、Web3/内容安全语境、面试/解释转化等维度设计评价标准；结合线上反馈、人工复核、申诉、质检进行闭环。
- **Result**：支撑 Agent 审核完整流程上线，并为后续 AI Agent 接口扩展打基础。
- **追问准备**：评估集如何采样？如何处理长尾风险？线上指标和离线指标如何对齐？

## 每个故事的 2 分钟模板

1. 背景：业务为什么重要，规模有多大。
2. 挑战：技术难点、组织难点或指标约束。
3. 方案：你作为 Owner/架构负责人做了哪些关键设计。
4. 结果：用数字说话。
5. 迁移：这个经验如何对应目标岗位。
